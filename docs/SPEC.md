# 機能仕様書

## サービス概要
**QiitaAI Watch** — QiitaおよびZenn上のAI関連記事を自動収集・閲覧できるWebサイト。

---

## フェーズ1（MVP）

### 画面一覧

#### 1. ホーム画面 (`/`)
- AI関連記事のカード一覧を表示する
- 初期表示は「新着順」で20件
- ページネーションで次ページを取得できる

#### 2. Aboutページ (`/about`)
- サイトの説明文を表示する静的ページ

---

### 記事カードの表示項目
| 項目 | 表示内容 |
|------|----------|
| タイトル | 記事タイトル（クリックでQiita元記事へ遷移） |
| 著者 | アイコン画像 + ユーザー名 |
| 投稿日 | YYYY/MM/DD 形式 |
| いいね数 | ハートアイコン + 数値 |
| タグ | 最大5件のタグバッジ |

---

### データソース
- **Qiita**: Qiita API v2 を直接使用
- **Zenn**: Zenn公開API (`https://zenn.dev/api/articles`) を使用

### ソースフィルター
- すべて（Qiita + Zenn の記事を統合表示）
- Qiitaのみ
- Zennのみ

---


### フィルタリング・ソート機能

#### UIレイアウト
- ホーム画面はサイドバー型レイアウトを採用する
  - 左カラム（幅: 固定 240px / md以上）: フィルターサイドバー
  - 右カラム: 記事一覧
  - md未満（モバイル）: サイドバーを上部に折りたたみ表示

#### タグフィルター
- タグ別フィルター（複数選択可）
- 対象タグ: `AI`, `機械学習`, `LLM`, `ChatGPT`, `DeepLearning`, `OpenAI`, `RAG`, `生成AI`
- 未選択時はすべてのAI関連記事を表示
- Zennはタグに対応するトピック名で検索

#### 期間フィルター
- 投稿日による絞り込み
- 選択肢: `全期間`（デフォルト）/ `今週` / `今月` / `3ヶ月以内`
- Qiita API の `created:>=YYYY-MM-DD` クエリを使用して実装

#### ソート
- 新着順（デフォルト）
- いいね数順

### ページネーション
- 1ページ20件
- 「前へ」「次へ」ボタン

---

### 非機能要件
- レスポンシブ対応（ブレークポイント: sm=640px, md=768px, lg=1024px）
- ダークモード対応（OSの設定に追従）
- ローディング中はスケルトンUIを表示
- APIエラー時はエラーメッセージを表示（リトライボタン付き）

---

## フェーズ2（拡張）

> フェーズ1完了後に着手する。

### 2-1. お気に入り＆カンバンボード機能

#### 概要
記事をお気に入りに追加し、カンバンボードで読書状況を管理できる機能。
データはすべて localStorage に保存する（バックエンド不要）。

#### 画面一覧

##### カンバンボード画面 (`/board`)
- お気に入りに追加した記事をカンバンボード形式で表示する
- 列（カラム）間でカードをドラッグ&ドロップで移動できる
- ヘッダーのナビゲーションからアクセスできる

#### お気に入り機能
- 記事カードにお気に入りボタン（♡ / ♥）を表示する
- クリックでお気に入りの追加/解除をトグルする
- お気に入り状態は localStorage に保存する
- お気に入りに追加した記事は自動的にカンバンボードのデフォルト列に配置される

#### カンバンボード機能

##### デフォルト列
- 初期状態で以下の3列を表示する:
  - **未読**（デフォルト列: お気に入り追加時にここに配置）
  - **読書中**
  - **読了**

##### カスタム列管理
- ユーザーが自由に列を追加できる（「+ 列を追加」ボタン）
- 列名の編集（インライン編集）ができる
- 列の削除ができる（列内のカードは「未読」列に移動）
- デフォルト3列は削除不可（リネームは可能）

##### カードの表示項目
| 項目 | 表示内容 |
|------|----------|
| タイトル | 記事タイトル（クリックで元記事へ遷移） |
| ソース | Qiita / Zenn バッジ |
| 著者 | ユーザー名 |
| いいね数 | ♥ + 数値 |
| お気に入り解除 | ✕ ボタン（お気に入りから削除） |

##### ドラッグ&ドロップ
- **ライブラリ**: `@dnd-kit/core` + `@dnd-kit/sortable`
- カードを列間でドラッグ&ドロップで移動できる
- 同一列内でのカード並び替えも可能
- ドラッグ中はオーバーレイ表示でカードのプレビューを表示する

##### データ永続化
- カンバンの状態（列定義・カード配置・並び順）はすべて localStorage に保存する
- localStorage キー: `qiita-ai-kanban-columns`, `qiita-ai-favorites`
- 保存タイミング: 状態変更のたびに即座に保存

#### 非機能要件
- レスポンシブ対応: モバイルでは列を横スクロールで表示
- ダークモード対応: 既存のダークモード設定に追従
- 空の列には「カードがありません」のプレースホルダーを表示

---

### 2-2. キーワード検索

#### 概要
記事タイトルをキーワードで絞り込むクライアントサイド検索機能。

#### 仕様
- ホーム画面の記事一覧上部に検索バーを表示する
- 入力されたキーワードで記事タイトルを部分一致フィルタリングする（大文字小文字を区別しない）
- 300msのデバウンスを適用し、入力のたびにフィルタリングを実行する
- 検索バーにはクリアボタン（×）を表示する
- キーワード検索はAPIフェッチ後のクライアントサイドフィルタリングとして実装する

---

### 2-3. 記事共有ボタン

#### 概要
記事カードにURLコピーボタンを追加し、ワンクリックで記事URLを共有できる機能。

#### 仕様
- 記事カードのいいね数の横にシェアボタン（リンクアイコン）を表示する
- クリックで記事URLをクリップボードにコピーする
- コピー成功時はアイコンをチェックマークに一時変更（2秒間）する
- `navigator.clipboard.writeText` を使用する

---

### 2-4. 読書統計ウィジェット

#### 概要
カンバンボードページに読書進捗の統計情報を表示するウィジェット。

#### 仕様
- カンバンボード画面の上部に統計バーを表示する
- 表示項目:
  - お気に入り総数
  - 各カラムの記事数
  - 読了率（読了カラムの記事数 / お気に入り総数 × 100）をプログレスバーで表示
- ダークモード対応

---

### 2-5. その他（未着手）
- [ ] OGP対応（SNSシェア用）

---

## 対象外（スコープ外）
- ユーザー認証・ログイン機能
- サーバーサイドの実装（フロントエンドのみ）
- 記事の投稿・編集機能

# GitHub Trending Viewer 仕様書

## サービス概要
**GitHub Trending Viewer** — GitHub Search APIを使ってトレンドのリポジトリを自動収集・表示するWebアプリ。フロントエンド完結。

---

## フェーズ1（MVP）

### 画面一覧

#### 1. ホーム画面 (`/`)
- トレンドリポジトリのカード一覧を表示する
- 初期表示は「今週作成された Star 数順」で20件

#### 2. Aboutページ (`/about`)
- サービス説明文を表示する静的ページ

---

### リポジトリカードの表示項目
| 項目 | 表示内容 |
|------|----------|
| リポジトリ名 | オーナー / リポジトリ名（クリックでGitHubへ遷移） |
| 説明文 | リポジトリのdescription |
| 言語 | プログラミング言語バッジ |
| Star数 | ⭐ + 数値 |
| Fork数 | 🍴 + 数値 |
| オーナーアイコン | アバター画像 |
| 作成日 | YYYY/MM/DD 形式 |

---

### データソース
- **GitHub Search API**: `https://api.github.com/search/repositories`（認証不要、60req/h）

---

### フィルタリング・ソート機能

#### 期間フィルター
- 選択肢: `今日`（デフォルト） / `今週` / `今月`
- GitHub Search APIの `created:>=YYYY-MM-DD` クエリで実装

#### 言語フィルター
- 対象言語: `すべて`（デフォルト）+ 主要言語一覧
- `language:xxx` クエリで実装

### ページネーション
- 1ページ20件
- 「前へ」「次へ」ボタン

---

### 非機能要件
- レスポンシブ対応（ブレークポイント: sm=640px, md=768px）
- ダークモード対応（OSの設定に追従）
- ローディング中はスケルトンUIを表示
- APIエラー時はエラーメッセージを表示（リトライボタン付き）
