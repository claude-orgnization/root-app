# /tdd-cycle — TDDサイクル実行スキル

## 説明
テスト駆動開発（RED → GREEN → REFACTOR）を実行するスキル。
タスクリストに沿って、テスト作成と実装を繰り返す。

## 使い方
```
/tdd-cycle                           # タスクリストの次のタスクを実行
/tdd-cycle "useAuth フックの実装"     # 特定タスクのTDDを実行
/tdd-cycle --all                     # 全タスクを連続実行
```

## 実行手順（1タスクあたり）

### RED — テスト作成
1. タスクの仕様・期待動作を確認する
2. テストファイルを作成する（対象ファイルと同じディレクトリに `*.test.ts(x)`）
3. 以下のテストを書く:
   - 正常系（主要ユースケース）
   - 境界値
   - 異常系（エラーケース）
4. テストを実行し、**失敗を確認する**:
   ```bash
   npm test -- --run <テストファイルパス>
   ```

### GREEN — 最小実装
1. テストを通す最小限のコードを書く
2. YAGNI — 今必要でないものは書かない
3. 型定義は `src/types/` に配置する
4. テストを実行し、**成功を確認する**:
   ```bash
   npm test -- --run <テストファイルパス>
   ```

### REFACTOR — 品質改善
1. 以下の観点で改善する:
   - 重複コードの排除
   - 命名の改善
   - 関数・コンポーネントの責務分離
   - 不要なコードの削除
2. テストを実行し、**引き続き成功することを確認する**:
   ```bash
   npm test -- --run
   ```
3. タスクを完了としてマークする

### 次のタスクへ
- 全タスク完了まで RED → GREEN → REFACTOR を繰り返す

## 重要なルール
- テストが RED でなければ GREEN に進まない
- テストが GREEN でなければ REFACTOR に進まない
- REFACTOR でテストが壊れたら即座に修正する
- `any` 型は絶対に使わない
- テストは振る舞いをテストする（実装の詳細をテストしない）
