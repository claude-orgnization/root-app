# /dev-flow — 開発フルフロー実行スキル

## 説明
要件定義からプルリクエスト作成まで、開発の全フローを一貫して実行するオーケストレーター・スキル。

## 使い方
```
/dev-flow "ユーザー認証機能を追加したい"
/dev-flow  # 対話的に要望をヒアリング
```

## 実行フロー

このスキルは以下の7つのフェーズを順番に実行します。
各フェーズの完了時にユーザーに確認を取り、承認後に次のフェーズへ進みます。

### Phase 1: 要件仕様定義
- `.claude/agents/spec-agent.md` のエージェントに従い仕様を策定する
- ユーザーへのヒアリング → `docs/SPEC.md` 更新 → 承認

### Phase 2: アーキテクチャ設計
- `.claude/agents/architect-agent.md` のエージェントに従い設計する
- `docs/ARCHITECTURE.md` と `docs/API.md` を更新する

### Phase 3: 技術スタック確認
- `CLAUDE.md` の既存スタックと照合する
- 新規ライブラリが必要なら提案・承認・インストールする

### Phase 4: タスク分解
- 仕様 + 設計からタスクを分解する
- TodoWrite でタスクリストを作成する
- 優先順位と依存関係を設定する

### Phase 5: TDDサイクル
- `.claude/agents/tdd-agent.md` のエージェントに従い実行する
- 各タスクに対して RED → GREEN → REFACTOR を実行する
- 全タスク完了まで繰り返す

### Phase 6: E2Eテスト（任意）
- ユーザーに E2E の必要性を確認する
- 必要であれば主要フローの E2E テストを作成・実行する

### Phase 7: プルリクエスト作成
- `.claude/agents/review-agent.md` のエージェントに従いPRを作成する
- テスト全通過を確認 → PR 作成 → URL を報告する

## フェーズ間の制御ルール
- 前のフェーズが完了・承認されるまで次に進まない
- 問題が発生した場合は該当フェーズに戻る
- ユーザーが特定フェーズのスキップを指示した場合はスキップ可能
- 各フェーズの成果物はドキュメントとしてコミットする
