# /spec-define — 要件仕様定義スキル

## 説明
ユーザーの要望を構造化された要件仕様書に変換するスキル。

## 使い方
```
/spec-define "ログイン機能を追加したい"
/spec-define  # 対話的にヒアリングを開始
```

## 実行手順

1. **引数の確認**: 引数があればそれを要望として使用。なければユーザーにヒアリングを開始する。

2. **既存仕様の確認**: `docs/SPEC.md` を読み込み、既存の仕様を把握する。

3. **ヒアリング**: 以下を明確にする質問をする:
   - 対象ユーザーは誰か
   - 実現したいことは何か
   - 成功の定義は何か
   - スコープの境界はどこか

4. **仕様書の作成/更新**: `docs/SPEC.md` に以下を追記する:
   ```markdown
   ## [機能名]

   ### 概要
   ### ユーザーストーリー
   ### 機能要件 (FR-xxx)
   ### 非機能要件 (NFR-xxx)
   ### 画面・UIフロー
   ### 受け入れ基準 (AC-xxx)
   ### 将来の検討事項
   ```

5. **ユーザー承認**: 仕様をユーザーに提示し、確認を得る。

## 出力
- 更新された `docs/SPEC.md`
