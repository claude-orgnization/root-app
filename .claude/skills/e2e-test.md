# /e2e-test — E2Eテスト実行スキル

## 説明
エンドツーエンドテストを作成・実行するスキル。主要なユーザーフローが正しく動作することを検証する。

## 使い方
```
/e2e-test               # 主要フローのE2Eテストを作成・実行
/e2e-test "ログインフロー" # 特定フローのE2Eテストを実行
```

## 実行手順

1. **テスト対象の特定**:
   - `docs/SPEC.md` の受け入れ基準を確認する
   - 主要ユーザーフローを洗い出す

2. **E2Eテスト作成**:
   - テストフレームワークの確認（Playwright / Cypress 等）
   - ユーザーシナリオに沿ったテストを記述する
   - テストファイルは `e2e/` ディレクトリに配置する

3. **テスト実行**:
   ```bash
   # 開発サーバーを起動してテスト実行
   npm run e2e
   ```

4. **結果確認**:
   - 全テストの成否を確認する
   - 失敗した場合は原因を調査し修正する

## テスト作成の指針
- ユーザーの操作フローをそのまま再現する
- セレクタは `data-testid` を優先する
- テスト間の依存をなくす（各テストは独立）
- ネットワークリクエストは必要に応じてモックする
